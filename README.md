# 🏦 Структура Базы Данных

## 📜 Обзор БД

### 1. 👤 `users` - Пользователи

Хранит информацию о зарегистрированных пользователях системы.

*   **Назначение:** Идентификация и аутентификация пользователей, определение их ролей.
*   **Ключевые поля:**
    *   `user_id` (SERIAL PRIMARY KEY): Уникальный номер пользователя (генерируется автоматически).
    *   `login` (VARCHAR(50) UNIQUE NOT NULL): Логин для входа (уникальный, обязательный).
    *   `password_hash` (VARCHAR(100) NOT NULL): Зашифрованный пароль пользователя (обязательный). **Пароли в открытом виде не хранятся!**
    *   `full_name` (VARCHAR(100)): Полное имя (необязательное).
    *   `role` (VARCHAR(10) CHECK (role IN ('client', 'admin'))): Роль пользователя (`client` или `admin`).
    *   `is_blocked` (BOOLEAN DEFAULT FALSE): Флаг блокировки пользователя (по умолчанию не заблокирован).
    *   `created_at` (TIMESTAMP DEFAULT CURRENT_TIMESTAMP): Дата и время регистрации (по умолчанию текущее время).
*   **Связи:** Является "родителем" для счетов (`accounts`) и истории входов (`login_history`).

---

### 2. 💼 `accounts` - Счета

Содержит информацию о банковских счетах пользователей.

*   **Назначение:** Учет денежных средств пользователей.
*   **Ключевые поля:**
    *   `account_id` (SERIAL PRIMARY KEY): Уникальный номер счета (генерируется автоматически).
    *   `user_id` (INTEGER NOT NULL REFERENCES users(user_id)): Ссылка на владельца счета в таблице `users` (обязательная).
    *   `account_number` (VARCHAR(20) UNIQUE NOT NULL): Уникальный номер банковского счета (например, "408...", уникальный, обязательный).
    *   `balance` (DECIMAL(15, 2) DEFAULT 0.00 CHECK (balance >= 0)): Текущий остаток на счете (по умолчанию 0.00, не может быть отрицательным).
    *   `currency` (VARCHAR(3) DEFAULT 'RUB'): Валюта счета (например, RUB, USD; по умолчанию RUB).
    *   `created_at` (TIMESTAMP DEFAULT CURRENT_TIMESTAMP): Дата и время создания счета (по умолчанию текущее время).
*   **Связи:**
    *   **Один ко многим (1:M)** с `users`: Один пользователь может иметь много счетов.
    *   Является "родителем" для карт (`cards`), транзакций (`transactions`) и переводов (`transfers`).

---

### 3. 💳 `cards` - Банковские Карты

Информация о банковских картах, привязанных к счетам.

*   **Назначение:** Управление банковскими картами.
*   **Ключевые поля:**
    *   `card_id` (SERIAL PRIMARY KEY): Уникальный номер карты (генерируется автоматически).
    *   `account_id` (INTEGER UNIQUE NOT NULL REFERENCES accounts(account_id)): Ссылка на счет в `accounts`, к которому привязана карта (уникальная, обязательная). **Гарантирует, что одна карта привязана только к одному счету, и один счет может иметь только одну основную карту (по такой логике).**
    *   `card_number` (VARCHAR(16) UNIQUE NOT NULL): Номер карты (уникальный, обязательный).
    *   `expiry_date` (VARCHAR(5) NOT NULL): Срок действия (например, "05/25", обязательный).
    *   `cvv` (VARCHAR(3) NOT NULL): CVV-код (обязательный). **Внимание:** В реальных системах CVV обычно не хранится!
    *   `is_blocked` (BOOLEAN DEFAULT FALSE): Флаг блокировки карты (по умолчанию не заблокирована).
    *   `pin_hash` (VARCHAR(100) NOT NULL): Зашифрованный PIN-код карты (обязательный).
*   **Связи:**
    *   **Один к одному (1:1)** с `accounts`: Один счет может иметь одну карту (в данной реализации).

---

### 4. 💸 `transactions` - Транзакции

Записи обо всех операциях по счетам (пополнения, списания, часть перевода).

*   **Назначение:** История движения средств по счету.
*   **Ключевые поля:**
    *   `transaction_id` (SERIAL PRIMARY KEY): Уникальный номер транзакции (генерируется автоматически).
    *   `account_id` (INTEGER NOT NULL REFERENCES accounts(account_id)): Ссылка на счет в `accounts`, по которому прошла операция (обязательная).
    *   `amount` (DECIMAL(15, 2) NOT NULL): Сумма операции (обязательная).
    *   `transaction_type` (VARCHAR(20) CHECK (transaction_type IN ('deposit', 'withdraw', 'transfer'))): Тип операции (`deposit` - пополнение, `withdraw` - снятие, `transfer` - участие в переводе).
    *   `description` (VARCHAR(255)): Описание (необязательное).
    *   `created_at` (TIMESTAMP DEFAULT CURRENT_TIMESTAMP): Дата и время транзакции (по умолчанию текущее время).
*   **Связи:**
    *   **Один ко многим (1:M)** с `accounts`: Один счет может иметь много транзакций.

---

### 5. ↔️ `transfers` - Переводы

Детализирует переводы денежных средств между двумя счетами.

*   **Назначение:** Логирование переводов как отдельных сущностей, отражающих двустороннюю операцию.
*   **Ключевые поля:**
    *   `transfer_id` (SERIAL PRIMARY KEY): Уникальный номер перевода (генерируется автоматически).
    *   `from_account_id` (INTEGER NOT NULL REFERENCES accounts(account_id)): Ссылка на счет-отправитель в `accounts` (обязательная).
    *   `to_account_id` (INTEGER NOT NULL REFERENCES accounts(account_id)): Ссылка на счет-получатель в `accounts` (обязательная).
    *   `amount` (DECIMAL(15, 2) NOT NULL CHECK (amount > 0)): Сумма перевода (обязательная, должна быть > 0).
    *   `commission` (DECIMAL(15, 2) DEFAULT 0.00): Комиссия за перевод (по умолчанию 0.00).
    *   `status` (VARCHAR(20) DEFAULT 'completed'): Статус перевода (например, `completed`, `pending`; по умолчанию `completed`).
    *   `created_at` (TIMESTAMP DEFAULT CURRENT_TIMESTAMP): Дата и время перевода (по умолчанию текущее время).
*   **Связи:**
    *   **Многие ко многим (M:M)** между `accounts` (через эту таблицу): Счет может быть и отправителем, и получателем во многих переводах.

---

### 6. 👣 `login_history` - История Входов

Записывает попытки входа пользователей в систему.

*   **Назначение:** Аудит безопасности, отслеживание активности входов.
*   **Ключевые поля:**
    *   `login_id` (SERIAL PRIMARY KEY): Уникальный номер записи (генерируется автоматически).
    *   `user_id` (INTEGER NOT NULL REFERENCES users(user_id)): Ссылка на пользователя в `users` (обязательная).
    *   `ip_address` (VARCHAR(15) NOT NULL): IP-адрес, с которого была попытка входа (обязательный).
    *   `login_time` (TIMESTAMP DEFAULT CURRENT_TIMESTAMP): Время попытки (по умолчанию текущее время).
    *   `is_success` (BOOLEAN NOT NULL): Была ли попытка успешной (обязательный).
*   **Связи:**
    *   **Один ко многим (1:M)** с `users`: Один пользователь может иметь много записей о попытках входа.

---

### 7. 💹 `currency_rates` - Курсы Валют

Хранит актуальные курсы обмена между различными валютами.

*   **Назначение:** Обеспечение конвертации валют при операциях или отображении балансов.
*   **Ключевые поля:**
    *   `rate_id` (SERIAL PRIMARY KEY): Уникальный номер записи курса (генерируется автоматически).
    *   `from_currency` (VARCHAR(3) NOT NULL): Исходная валюта (код, например, USD; обязательная).
    *   `to_currency` (VARCHAR(3) NOT NULL): Целевая валюта (код, например, RUB; обязательная).
    *   `rate` (DECIMAL(10, 4) NOT NULL): Обменный курс (обязательный).
    *   `updated_at` (TIMESTAMP DEFAULT CURRENT_TIMESTAMP): Время последнего обновления курса (по умолчанию текущее время).
    *   `UNIQUE (from_currency, to_currency)`: Гарантирует уникальность пары валют для курса.
*   **Связи:** Справочная таблица, используется другими частями приложения.

---

## ⚡ Индексы

Для ускорения часто выполняемых запросов (например, поиск счетов пользователя или транзакций по счету) созданы следующие индексы:

*   `idx_accounts_user_id` на `accounts(user_id)`
*   `idx_transactions_account_id` на `transactions(account_id)`
*   `idx_transfers_from_account` на `transfers(from_account_id)`
*   `idx_transfers_to_account` на `transfers(to_account_id)`
*   `idx_login_history_user` на `login_history(user_id)`
